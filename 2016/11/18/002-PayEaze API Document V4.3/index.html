Solution

PayEase online payment mainly includes the merchant transaction order submission process and the bank order confirmation process.

The merchant transaction order submission process:

when the merchant server forms the final order and the consumer selects the PayEase mode and the consumer selects the PayEase In the platform to select the bank for online payment transactions, the interface program will directs the payment information in the consumer order to the PayEase platform.

Bank order confirmation process:

When the transaction is completed, PayEase platform can notify the merchant bank in two ways to confirm the messageof the order, the merchant needs to prepare the corresponding program to receive the relevant payment confirmation message returned by the bank according to the actual business needs of two options below (either or all) , and to carry out subsequent business operations by judging the message.

  1. Foreground page link: The result (parameters) of the transaction of the payment order is to connect to the form of pages sent to the merchant . The foreground here means that this message passing process is visible to cardholder consumers.

    This approach is characterized by: return to the real-time parameters. but there are several individual payment channel do not support this approach, such as network interruption or cardholder does not operate as required by the merchant. this approach may let merchant missing a order (means that merchant can not receive a bank payment confirmation message). Once it happens, the merchant can only receive them through the background interface program, or log on to The PayEase Merchant Management System for confirmation.

    Note: In this way, the url of the receiving program is specified by the value of the parameter v_url when the merchant submits the order without prior notice to the PayEase platform. If there is any change, the value of v_url can be modified before the order message is submitted.

  2. Backgrount interface program: The PayEase platform forwards confirmation messages to the merchant. The way, PayEase platform server communicates with the merchant server is invisible to the cardholder.

    This approach is characterized by: support all payment channel on the platform, and in the process of communication with the merchant if the network failure, this approach supports automatic retransmission in the first 24 hours.

    Note: In this way, the url of the receiving program needs to be determined by the merchant. When the merchant ID is activation, This imformation is filled in the “Order Confirmation Return Address” column of the initial form, and the PayEase platform administrator will follow the initial form content to activate the ID. If the changes are made after the activating, the administrator must be notified to confirm the changes.

PayEase will immediately notify the merchant with transfer results when transfer is successful. The url of this interface is confirmed by the merchant and is entered in the “Transfer Confirmation Notification Interface” column of the initial form. After the activating of the merchant ID, if necessary, prior notice to the administrator to confirm changes.

Note: Notification to the merchant transfer process and online payment transactions are two relatively independent process, transfer work is done by the PayEase settlement center staff line, after informing the bank to transfer work, through the “PayEase notify the merchant transfer results interface” Send all the order IDs of this transfer to the merchant. Because the interface to send the information is only played a notice role, does not affect the business of the actual transfer work, so businesses can choose according to their actual needs to use this interface.

3.The order payment result is returned in the background interface

1) Interface description：

  • PayEase to Merchant
  • The PayEase platform sends the payment result of the order to the merchant, and the merchant returns the receiving condition.

Note:
This interface and the subsequent transfer result return interface are to be provided by the merchant when the merchant is settled in. In the “Order Payment Return Interface” column of the initial order, or in the form of company name, merchant number, contact, URL of both interfaces Sent to the platform of the customer service mailbox.

  • The PayEase platform will return the payment result for one or more orders at a time, and will be sent as an order group when it returns to multiple order results: The PayEase platform sends the payment results to the merchant in real time for each order. If an order has not been successfully sent (if there is a network outage or the order md5 is incorrect) but the payment has been successfully made, the order will be sent the next time the order is sent. If the delivery is not successful again, Repeat the same day. Orders for failed payments are sent only once.
